Work Experience

Facebook Internship

This was a 12–week internship at Facebook's headquarters in Menlo Park, California. I was on the Facebook Chat team, working on a redesign of the Messages user interface. The new interface made heavy use of JavaScript and ajax to avoid full page reloads, improving responsiveness. It displayed the page in a two–column layout which allowed users to see their inbox and specific conversations simultaneously.

My most significant contributions were in the search feature. The original Messages interface reloaded the page on every search, hampering user experience. I updated the frontend to search the conversations available locally while waiting for the server to respond with a full set of results, providing instant feedback to queries.

I also contributed to several other features, including highlighting results in full–text searches to indicate relation to the query, resolving compatibility issues with legacy browsers, implementing an interface to move between full–text search results, and displaying useful messages to the user in unusual scenarios.

The new interface was launched to Facebook employees near the start of the summer, and began rolling out to the public on August 22nd, 2012.

Software Design Group UROP

I did this urop project under the supervision of Prof. Daniel Jackson at mit. I was involved in Rubicon, a library used for static verification of Rails web apps. Rubicon uses Ruby's flexibility as a programming language to change the execution model of client code to build a model to analyze.

Rubicon's attempts to redefine parts of the language were stymied by the existence of built-in keywords like “if”, “and”, and “or”; these could not be redefined like method calls. I designed and implemented code to make keyword redefinition possible by walking the abstract syntax tree and rewriting expressions, with the help of libraries including ParseTree, ruby_parser, and sexp_processor.

My project was modular enough to be extracted out into its own library: virtual_keywords, available at this Github repository. virtual_keywords allows developers to redefine the behavior of Ruby code that uses built–in keywords. It does not expect developers to understand the intricate details of Ruby's grammar.

Google Internship

I was on the Docs Engineering Productivity team, working on JsTestDriver, a framework for running unit and integration tests on JavaScript applications. JsTestDriver is being used by the Docs and Analytics teams, among others. It's also being used outside of Google.

I added functionality to allow clients to embed the test runner in the browser along with the system under test. The existing method instead started a separate server, and ran the test runner on the server. I designed and implemented a menu user interface, and reworked the test runner so it could be compiled by the Google Closure Compiler along with the rest of the system under test.

My contributions to JsTestDriver allowed it to provide an alternate workflow with a faster feedback loop in scenarios where cross-browser compatibility does not need to be verified. (e.g. calls to libraries like jQuery which handle browser differences internally)

IsisWorld UROP

IsisWorld is an open-source 3D simulator designed for research in commonsense ai systems. It puts actors into a kitchen environment, where they are given the goal of accomplishing tasks like making toast. The challenge is in training them to perform these tasks without overspecialization or scripting. My job involved fine-tuning the simulation environment and the scenarios, and giving the system a layer of polish before we advertise it more.

I also developed a related tool that allows users to tweak and optimize 3D models in the .egg file format (used by the Panda3D library). This is being used to prepare models for IsisWorld, but it also has utility for other projects that use 3D graphics with .egg files.

A more detailed summary of IsisWorld can be found here and its code is hosted in this Github repository.

Infosys Internship

I developed a constraint evaluation algorithm as part of Infosys' ftgen software which automatically generates test data for software systems. It reads a predicate defined in a constraint–specification language and decides how to satisfy the predicate. My code sped up the software by more than an order of magnitude for a class of inputs that could be solved via boundary value analysis and equivalence partitioning. Without my contribution, all inputs had to be processed by a genetic algorithm which could solve any possible input, but was significantly slower.

This project used Java and Eclipse. It involved both software engineering and computer language engineering, as I had input in the design of the constraint specification language.

Intern at University of Maryland

I developed algorithms to analyze problems in classical cryptology under the mentorship of Dr. William Gasarch , a Computer Science professor at the University of Maryland. These algorithms attempted ciphertext–only attacks on texts enciphered through various means, including random polyalphabetic ciphers and polygraphic ciphers with large enciphering matrices. This project culminated in the development of a suite of programs that analyzed various ciphers and a presentation of my findings related to the feasibility of applying modern computing methods to classical ciphers.
